# Drevalis Security Fixes

**Date:** 2026-04-30 (audit branch `audit/2026-04-29`)
**Scope:** Fixes for HIGH/MEDIUM/LOW security findings from the post-March-2026
re-audit. See `audit/security.md` on the audit branch for the full
finding catalogue.

---

## HIGH Fixes

### F-S-01: `cryptography` 46.0.5 carries two CVEs
**File changed:** `pyproject.toml`

- Bumped pin to `cryptography>=46.0.7` to close CVE-2026-34073 and
  CVE-2026-39892. The package underpins Fernet encryption, OAuth token
  encryption, and Ed25519 license-JWT signing/verification — every
  finding fix was gated on landing the bump first.

### F-S-04: TikTok OAuth callback never validated `state`
**File changed:** `src/drevalis/api/routes/social.py`

- Reject `/tiktok/callback` requests with empty / unknown / replayed
  `state`; redirect with `tiktok_error=invalid_state` so the frontend
  can route to the activation wizard.
- Switched the Redis lookup to atomic `getdel` (matches the YouTube
  callback pattern) so a parallel callback can't reuse the same state.
- Closes the gap that previously let an attacker who tricked an
  operator into following a crafted callback URL connect their own
  TikTok account to the victim's install.

### F-S-05: Backup tar extraction lacked symlink/special-file guard
**File changed:** `src/drevalis/services/backup.py`

- `_safe_extract` now passes `filter='data'` to
  `tarfile.extractall`, rejecting symlink, hardlink, device, and FIFO
  members. The pre-walk path-traversal check is preserved as
  defence-in-depth.
- Closes Bandit B202 HIGH and the symlink-pivot vector against
  operator-uploaded backup archives.

## MEDIUM Fixes

### F-S-02: `anthropic` 0.86.0 carries two CVEs
**File changed:** `pyproject.toml`

- Bumped pin to `anthropic>=0.87.0` to close CVE-2026-34450 and
  CVE-2026-34452. Used by `AnthropicProvider` in the LLM service for
  outbound LLM calls.

### F-S-08: Session cookie HMAC reused `ENCRYPTION_KEY`
**Files changed:** `src/drevalis/core/config.py`,
`src/drevalis/api/routes/auth.py`, `.env.example`

- Added `SESSION_SECRET` env var and a new
  `Settings.get_session_secret()` helper that prefers it and falls
  back to `ENCRYPTION_KEY` for backward compatibility. Production
  installs should set `SESSION_SECRET` independently so a Fernet-key
  leak (backup compromise, log-capture of `decrypt_value` exception
  text) does not also yield session-token forgery for the entire fleet.
- Added `COOKIE_SECURE` env var (default `false`) so production
  installs behind HTTPS can opt into the `Secure` cookie flag without
  breaking local-HTTP dev.

## LOW Fixes

### F-S-13: License-server admin endpoints have no rate limit
**Files changed:** *(none in this repo)*

- Surfaced as a follow-up: `license-server/` lives in a separate
  gitignored repo. Apply the existing `Depends(rate_limit_ip(...))`
  pattern (used on `/activate`, `/portal`, `/checkout`) to the admin
  endpoints there. With a leaked admin token, an unrate-limited
  scanner can enumerate every license, mint preview JWTs, and revoke
  licenses at full speed.

### F-S-15: Bandit B324 SHA-1 false positive
**File changed:**
`src/drevalis/services/audiobook/_monolith.py`

- Pass `usedforsecurity=False` to `hashlib.sha1` for the audiobook
  title-card filename slug (8-char collision avoider, not an
  integrity check). Silences the scanner without an inline ignore.

### F-S-19: WebSocket auth token in query string can leak via proxy logs
**File created:** `docs/security/websocket-token-logging.md`

- Documented the operational risk (most reverse-proxy access logs
  record full URLs by default) and provided per-proxy scrubber recipes
  for nginx, Caddy, and the actual NPM fronting host.
- Documented the explicit decision **not** to switch to
  `Sec-WebSocket-Protocol` or first-message auth — both move the risk
  rather than removing it. Browsers cannot set `Authorization` on
  `WebSocket()`, so the query-string approach concentrates the leakage
  to one predictable place that operators can scrub.

### F-S-NN (housekeeping): widened CHECK constraints + new env vars

These are surfaced as follow-ups in `docs/ops/techdebt.md`:

- Multi-version `ENCRYPTION_KEY_V*` env-var auto-loading is not
  wired into `Settings`. Documented in techdebt §4.
- `core/http_retry.py` / `services/llm/_monolith.py` /
  `services/audiobook/_monolith.py` each carry their own retry
  control flow. A `compute_backoff_seconds()` helper is the only
  shared math worth extracting; documented in techdebt §5.

## Not fixed (intentional, per audit `Don't fix` list)

- `validate_safe_url_or_localhost` permits localhost / private ranges —
  required for local-first deployment with ComfyUI on `127.0.0.1` or
  on a NAS LAN IP.
- `/storage/*` is exempt from the license gate — operators always
  deserve their own past output.
- `/api/v1/auth/mode` is intentionally public so the `LoginGate`
  component can pre-render before any auth state exists.
- WebSocket token in query string — see F-S-19 above; the alternative
  (header echo on handshake) has the same logging concern.
- No HSTS / CSP — local-first single-tenant; CSP would break the React
  dev server.
- Cookie `secure=False` default — local-HTTP dev needs it; production
  flips via `COOKIE_SECURE=true`.
