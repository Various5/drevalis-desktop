## Phase 3 — Global UX polish                                                                                                                                                                                                       
Goal: every screen feels finished, not alpha.                                                                                                                                                                                       
                                                                                                                                                                                                                                    
- **Empty states**: every list view (Episodes, Series, Audio Studio, Assets, Characters, Calendar, Jobs, Channels) gets an illustration + one-line description + primary CTA                                                        
- **First-run dashboard**: until at least one series exists, the Dashboard shows a "Start here" card linking to Series creation, plus three example series templates the user can fork                                              
- **Command Palette**: expand beyond routes — add actions ("Create episode in <series>", "Connect YouTube", "Open last edited episode", "Toggle LAN API", "Restart worker"). Index dynamically from the current project.            
- **Global undo**: a Cmd/Ctrl+Z stack that survives across routes for the last N destructive actions (delete episode, delete scene, cancel job, disconnect channel). Toast with "Undo" button after every such action.              
- **Notifications**: replace one-off toast usage with a unified notification centre in the header (bell icon, list of recent events from the WS channel — completed renders, failed jobs, publish results). The Active Jobs popover 
 stays for *running* state; the centre owns *historical* state.                                                                                                                                                                     
- **Activity Monitor**: keep, but make it dismissable per-session and remember position                                                                                                                                             
- **Shortcut overlay**: regenerate from a single source of truth (`shortcuts.ts`) so the cheat sheet is never out of sync with reality                                                                                              
- **Header**: drop the page title on routes that already show one in their content header (kill duplicate H1s); keep on routes that don't                                                                                           
- **Loading**: every async route gets a skeleton, not a spinner; spinners only for in-button waits                                                                                                                                  
                                                                                                                                                                                                                                    
Acceptance: navigating a fresh install with zero data feels guided, not broken.
