## Phase 4 — LAN API Access hardening + dangerous-action treatment                                                                                                                                                                  
Goal: the newest feature is also the riskiest — make it foot-gun-proof.                                                                                                                                                             
                                                                                                                                                                                                                                    
- Promote LAN API Access to its own Settings group **Network**, not buried under System                                                                                                                                             
- Treatment of the toggle:                                                                                                                                                                                                          
  - OFF state: green "Local only" pill, single-line explainer                                                                                                                                                                       
  - ON state: amber "Exposed on LAN" pill with the resolved URLs (filtered per Phase 0), the token (masked, copy button, "rotate" button), and a "Disable" button                                                                   
  - Before flipping ON: a modal with a clear warning, a copy of the threat model in one paragraph, and a typed-confirmation ("type EXPOSE to enable") — Cloudflare-style destructive confirm                                        
- Restart-required banner becomes a coloured strip at the top of the route + a "Restart backend" button (Tauri command that gracefully stops the bundled `drevalis.exe` and respawns it)                                            
- Audit log: every toggle event + token rotation written to System Log with timestamp + source                                                                                                                                      
- Apply the same destructive-confirm pattern to: Delete series (with N episodes), Delete account, Disconnect channel (with M scheduled posts), Reset database, Wipe storage                                                         
                                                                                                                                                                                                                                    
Acceptance: a user cannot enable LAN exposure by accident; the on-state communicates risk at every glance.
