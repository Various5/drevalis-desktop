Windows 10 · 11 · 64-bit

Native installer

An NSIS .exe — installs as a regular user (no admin prompt needed). FFmpeg and Redis are bundled inside the install directory. The first launch creates a SQLite database under %LOCALAPPDATA%\Drevalis. macOS and Linux are not supported at this time.

Get the .exe

What the installer puts on your disk

  1. 1The app, FFmpeg, and a bundled Redis sidecar go into %LOCALAPPDATA%\Drevalis Creator Studio — about 250 MB on disk.
  2. 2Your SQLite database, logs, and generated episodes go into %LOCALAPPDATA%\Drevalis (separate from the program files so an uninstall doesn't nuke your work).
  3. 3A Start-menu shortcut and a tray icon. The app keeps running in the tray when you close the window so background generation doesn't get killed.
  4. 4That's it. No Docker, no PostgreSQL, no Python — everything's bundled and self-contained.

System requirements

Minimum

Runs, but slow on big jobs

  • — Windows 10 (22H2+) or Windows 11
  • — 4 CPU cores, 8 GB RAM
  • — 5 GB free disk for the app + a few episodes
  • — GPU optional — needed only for ComfyUI image/video generation
Recommended

Smooth on Shorts and long-form

  • — 8+ CPU cores, 16 GB+ RAM
  • — NVIDIA GPU with 12 GB+ VRAM (for local ComfyUI)
  • — 50 GB+ NVMe for episode storage
  • — Or rent a cloud GPU from RunPod / Vast.ai (Pro plan) and the app itself runs fine on a laptop

The app itself is light. The heavy lifting is whatever LLM, TTS, and ComfyUI workflow you point it at — they decide your hardware ceiling, not Drevalis.

External tools you point it at

Drevalis orchestrates these — they aren't bundled. You install them once and Drevalis remembers the URL.

ComfyUI

Scene image and video generation. Default URL http://localhost:8188; the app discovers it the first time you launch.

After you install

  1. 1Launch from the Start menu. The app window opens after a few seconds (the bundled Redis + worker need to spin up).
  2. 2Paste your license key from the purchase email into the activation screen. It exchanges with license.drevalis.com for a signed token and unlocks the app.
  3. 3Settings → ComfyUI Servers → add your ComfyUI URL (usually http://localhost:8188).
  4. 4Settings → LLM Configs → add your LM Studio / Ollama / Claude endpoint and pick a default model.
  5. 5Settings → Voice Profiles → pick a default TTS voice (Edge or Kokoro are free and ship locally).
  6. 6Create your first series and generate an episode.

Updates

In-app, one click: Settings → Updates → Check for updates. The Tauri auto-updater downloads the signed installer, verifies the Ed25519 signature against the public key embedded in your install, and re-launches the new version. No reinstall, no migrations to run manually.

Prefer a manual update? Download the latest .exe from GitHub Releases and run it. Your data lives in %LOCALAPPDATA%\Drevalis — completely separate from the program files, so reinstalling won't touch anything you've made.

Want to verify the download?

Every release ships three artifacts next to each other on GitHub: the installer, a detached .sig (Ed25519 minisign), and a latest.json updater manifest. If you want belt-and-braces verification before running an .exe:

# 1. Download the installer + .sig from the same release page.
# 2. Verify with minisign and the public key from:
#    https://github.com/Various5/drevalis-desktop/blob/main/.tauri-keys/drevalis-updater.key.pub

minisign -Vm "Drevalis Creator Studio_*_x64-setup.exe" \
         -P "RWQ25V8RbLTQAErpTcxm7HBW6OojHEAQzHLEF4tkAXtOXp/LbxrK5jZN"

The in-app auto-updater does this for you every time it pulls a new version — these manual steps are only if you want to verify before the first install.
